### 🏢 HireNode — AI-Powered Hiring Platform
> *A production-grade, microservices-based recruitment platform that transforms how companies hire and candidates grow.*

**What it does:** HireNode connects job seekers with employers through an intelligent hiring pipeline — from AI-powered resume analysis and role recommendations to seamless applications and payments — all backed by a resilient, event-driven microservices architecture.

```
Architecture Overview:
┌─────────┐    ┌─────────┐    ┌──────────┐    ┌─────────┐    ┌─────────┐
│  Auth   │◄──►│  Users  │◄──►│   Jobs   │◄──►│Payment  │◄──►│  Util   │
│ Service │    │ Service │    │ Service  │    │ Service │    │ Service │
└────┬────┘    └────┬────┘    └────┬─────┘    └────┬────┘    └────┬────┘
     │              │              │                │               │
     └──────────────┴──────────────┴────────────────┴───────────────┘
                                   │
                              Apache Kafka
                         (Async Event Bus — decoupled inter-service comm)
```

**⚙️ Key Engineering Decisions:**
- **Decomposed into 5 independent services** (Auth, Users, Jobs, Payment, Util) — each deployable and scalable in isolation
- **Apache Kafka** as the async event bus — services emit events instead of direct HTTP calls, preventing cascading failures
- **Neon Serverless PostgreSQL** — instant connection pooling without managing DB infrastructure, ideal for a microservices setup
- **Gemini AI integration** — analyzes resumes against job descriptions and outputs a ranked match score, personalized career guidance, and role-fit recommendations
- **Cloudinary** for optimized asset storage (resumes, profile images) — offloads file handling from core services

**Tech:** `Next.js` `TypeScript` `PostgreSQL (Neon Serverless)` `Apache Kafka` `Redis` `ShadCN` `Cloudinary` `Gemini AI`

### 💬 Chatify — Production-Ready Real-Time Chat
> *A scalable, secure 1-to-1 messaging system engineered with anti-spam architecture, dual OTP auth, and cursor-based pagination.*

**What it does:** Chatify is a full-featured real-time chat application that solves three hard problems most chat apps ignore: preventing message spam at scale, loading long chat histories efficiently, and providing flexible multi-channel authentication.

**⚙️ Key Engineering Decisions:**

**Two-Layer Redis Rate Limiting (Anti-Spam Architecture):**
```
Incoming Message
       │
       ▼
┌─────────────────────────────────────┐
│  Layer 1: Global User Limit         │
│  key: msg:{userId}                  │
│  → Max 30 msg / 60s globally       │
└──────────────────┬──────────────────┘
                   │ PASS
                   ▼
┌─────────────────────────────────────┐
│  Layer 2: Per-Conversation Limit    │
│  key: msg:{userId}:{conversationId} │
│  → Max 15 msg / 10s per chat       │
└──────────────────┬──────────────────┘
                   │ PASS
                   ▼
            Persist to MongoDB
              (no wasted writes)
```

- **Dual OTP Authentication** via Twilio (SMS) + SendGrid (Email) — users choose their channel
- **Cursor-based pagination** using MongoDB `_id` as cursor — returns `messages`, `nextCursor`, `hasMore`. Loads in reverse-chronological chunks; scales gracefully for 10k+ message histories without offset drift
- **MongoDB indexing** on conversation + timestamp fields — dramatically reduces query time on high-volume chat rooms
- **Upstash (Cloud Redis)** — managed Redis with no cold starts, used for both rate limiting and session state

**Tech:** `React.js` `Node.js` `Express.js` `Socket.IO` `MongoDB` `Redis (Upstash)` `JWT` `Twilio` `SendGrid` `Cloudinary` `Zustand`

### 🔢 DigiQ — Smart Digital Queue Management System
> *Eliminates physical waiting lines by replacing them with a real-time, QR-accessible digital queue — built for salons, clinics, cafés, and banks.*

**What it does:** DigiQ lets any service provider go digital in minutes. They register, add their services with estimated durations, and get a QR code to display at their shop. Customers scan it, join the queue from their phone, and see a dynamically calculated wait time — no app download, no ticket, no confusion.

**⚙️ Key Engineering Decisions:**

**Dynamic Wait Time Algorithm:**
```
User joins queue at position N
       │
       ▼
Fetch all users ahead (positions 1 to N-1)
       │
       ▼
For each user ahead → get their chosen service duration
       │
       ▼
Total Wait = Σ (service_duration of all users ahead)

Example: [User A: 4min] + [User B: 4min] + [User C: 5min] = 13 min ETA
```

- **Real-time queue updates** — admin marks served/skipped → all connected clients update instantly
- **Automatic QR Code generation** per organization on registration — printable, shareable, zero friction for end-users
- **City-based filtering** — customers can browse and join queues from nearby organizations
- **Zero-login queue joining** for customers — reduces drop-off; authentication only required for org management

**Tech:** `React.js` `Node.js` `Express.js` `MongoDB Atlas` `Tailwind CSS` `QR Code (npm)` `REST API`
